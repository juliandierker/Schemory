# Schemory Brand Assets

## Logo Variants

### App Icons (100x100, rounded square)
- **logo-light.svg** - Purple symbol on white background (light mode)
- **logo-dark.svg** - Purple symbol on dark background (dark mode)
- **logo-purple.svg** - White symbol on purple background (brand color mode)

### Favicon (32x32)
- **favicon.svg** - Purple symbol, transparent background (light mode)
- **favicon-dark.svg** - Purple symbol on dark background (dark mode)

### Full Logo
- **logo-full.svg** — Complete logo with "schemory" wordmark (light backgrounds)
- **logo-full-dark.svg** — Complete logo with "schemory" wordmark (dark backgrounds)

## Color Palette
- **Brand Purple**: `#7878F8` (Periwinkle violet)
- **Dark**: `#1A1A2E` (Dark navy-black)
- **Light**: `#FFFFFF` (White)

## Two-Tone Design
The symbol uses a two-tone scheme to give it visual character:
- Left `{` — always in brand purple `#7878F8`
- Right `}` — adapts to background: dark `#1A1A2E` on light bg, white `#FFFFFF` on dark/purple bg
- `=` — matches the left brace color

## Usage
- Use `logo-light.svg` for light theme app icons
- Use `logo-dark.svg` for dark theme app icons
- Use `logo-purple.svg` for purple/brand theme app icons
- Use `favicon.svg` and `favicon-dark.svg` for browser favicons (auto-switches based on OS theme)
- Use `logo-full.svg` for dashboard header or promotional materials

## Symbol Design
The `{ = }` symbol represents:
- `{}` - Code/type definitions
- `=` - Assignment/equality (TypeScript/JavaScript)
- Combined: Type definitions and equality, core to Schemory's purpose
